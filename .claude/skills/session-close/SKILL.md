# Session Close

Leave the next session a clean handoff.

1. Summarize what changed, what completed, what remains, and unresolved risk.
2. Propose updates to each touched project's `03-status/status.md`: current state, one next
   action, blockers, decisions, and a dated activity line.
3. Propose root `03-status/` updates only for cross-project state, tasks, or decisions.
4. Promote knowledge only when it is durable, reviewed, and has provenance. Keep it in the
   project first unless it applies broadly.
5. Show the short update plan and ask before writing ambiguous or sensitive material.
6. Verify no secrets or raw personal content entered status or knowledge.
7. End with the single best next action.

Do not publish, send, sync, commit, or change external systems unless separately requested
and approved.
