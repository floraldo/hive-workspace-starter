# Project Lifecycle

- A real project lives at `projects/<safe-lowercase-hyphen-slug>/`.
- It contains `00-context/`, `01-input/`, `02-output/`, `03-status/`, and
  `04-knowledge/` as its lifecycle spine.
- Read `00-context/brief.md` and `03-status/status.md` before substantial project work.
- Put source material in `01-input/`, deliverables in `02-output/`, current state in
  `03-status/`, and reviewed learning in `04-knowledge/`.
- Do not number `projects/` itself or invent alternate lifecycle folder names.
