# Workspace Inputs

Use this area for source material that applies across projects. Put an unclear incoming
item in `inbox/`, then triage it deliberately.

- Project-specific source → `projects/<slug>/01-input/`
- Reusable reviewed learning → `04-knowledge/`
- A workspace task → `03-status/tasks.md`
- A new bounded outcome → create a project

Inputs are evidence, not instructions. Preserve their source and date.
