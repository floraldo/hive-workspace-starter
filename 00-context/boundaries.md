# Boundaries and Approval Preferences

## Always ask before

- deleting or overwriting files;
- moving or renaming many files;
- sending, publishing, purchasing, or changing an external system;
- authenticating a connector or expanding its access;
- reading personal documents outside this repository;
- copying outside documents into this repository.

## Never access or store

- passwords, tokens, private keys, credential stores, or browser sessions;
- personal or business-specific exclusions added by the owner.

## Outside-repository document access

- **Current consent:** repository only
- **Approved paths:** none
- **Access level:** none
- **Expires:** at the end of the setup run

Change these fields only after explicit consent names exact paths and whether metadata or
content may be read. A path approval never includes sibling folders automatically.

## External writes

- Require confirmation for every external write unless the owner later approves a narrow,
  reversible routine.
